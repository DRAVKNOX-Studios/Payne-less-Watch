# Contributing to Payne-less: Typer

Thanks for your interest in contributing! Please read the following
guidelines before submitting anything.

---

## Code Contributions

### AI-Assisted Code
AI tools are permitted but with conditions:
- You **must disclose** any AI-assisted code in your pull request
- AI should be used in an **assistive capacity**, not as the sole author
- You are expected to **fully understand** what the AI wrote
- You are **responsible** for catching and fixing any AI-generated
  mistakes, reviewers will not do this for you!

### Code Quality
- All submitted code must be **tested by you** before opening a PR
- The code must do exactly what you claim it does
- It must not break any existing functionality
- It must not introduce new bugs

### File Standards
- **300 line hard limit** per file, no exceptions!
- **No inline code comments** - explain your changes in the PR instead
- New files must follow the same structure as existing ones

### Core Philosophy
All contributions must align with what this keyboard stands for.
Features or changes that compromise any of the following will be
rejected outright:

- 🔒 **Private** - no internet calls, no telemetry, no tracking
- 📴 **Offline** - no cloud dependencies of any kind
- 💾 **Local** - all data stays on the device
- ⚡  **Fast** - no sluggish UI, no heavy libraries
- 📱 **Low-end friendly** - must run well on 1GB RAM devices

If your contribution moves away from any of these, it's not a fit
for this project regardless of how good the code is.

---

## Bug Reports & Issues

Found a bug? Please open an issue using the bug report template.

Your report should include:
- A clear description of the problem
- Steps to reproduce it
- Expected vs actual behavior
- Your Android version and device (model optional)
- Whether you're on a custom ROM

> Feature requests are also welcome via issues. Use the feature
> request template and explain how it fits the core philosophy above.

---

## Pull Request Checklist

Before submitting, make sure:

- [ ] Code is tested and working
- [ ] No existing functionality is broken
- [ ] File(s) stay under 300 lines
- [ ] No inline comments in code
- [ ] AI usage is disclosed (if applicable)
- [ ] Changes align with the core philosophy
- [ ] PR description explains what, why, and how

*Payne-less: Watch is about doing one thing well.
Keep it fast. Keep it private. Keep it lean.*
